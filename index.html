一只猫的记忆

2023-11-23

今天换了一个微信头像，头像内容的主人公是一只猫。其实很早之前就用过这张图片作为头像，我总是很喜欢用一些从前用过的东西，会换回之前用过的头像；听歌也是如此，用一阵网易云音乐之后，我就会换回QQ音乐，再用一阵QQ音乐之后，再换回网易云音乐。无论是头像也好，音乐也罢，它们都已经不再是单纯的头像和音乐，而是随着时光演变成了最初的记忆的载体。每当我重温它们的时候，这张图片，或者这首歌曲随之而来的记忆就会浮现在眼前，有时候竟然觉得这些东西变成了一扇门的开关，打开它们就仿佛回溯到当时的场景。这种感觉是很玄妙的，总是让我沉迷其中，无法自拔。

每当我看到这张图片的时候，总有一种异样的感觉。那种感觉就像，我变成了图片中的猫，在落日的余晖下，当然也有可能是在朝阳下，露出嘴角略微上扬的洒脱的笑，挥挥手，向记忆告别，告别过去的一些人，告别过去的一些事，告别过去的一些记忆。随后，猫转头，迎着不论是朝阳的晨光还是落日的余晖，踏上一段新的旅程，而过去的一些事物，已经可以释怀，永远沉寂在了记忆中。

说到记忆，就回想起那日和舍友的谈话。其实我们所谓的记忆，也不过是自己对自己的不断欺骗罢了，那些最原本，最真实的记忆早就已经不复存在，我们现在所拥有的，只不过也是我们自己对自己暗示所得到的”修正“的记忆。因此想来，自己那些所谓美好的一些回忆，是否真的就是如此呢，我们不得而知，没有人知道答案。

Robocopy迁移hexo文件保证博客更新时间不变

2023-11-20

前言

说起来已经有好几篇博客的源头都是因为换了新电脑，这篇也是一样。因为这个博客是用hexo框架搭建的，之前的博客更新全在我的笔记本电脑上面，但是换了新台式电脑之后就需要把 hexo文件全部迁移过来，在新电脑上进行博客的撰写可更新。但是当我简单的把需要的文件全部拷贝到新电脑上的时候，发现我的博客更新时间全变成了拷贝的时间。

这是因为普通的拷贝会把文件的修改时间设置为拷贝的时间，而Hexo默认是以文件的修改时间作为文章的更新时间的。因此可以看到上面所有的博客的更新时间都被错误的同样设置为11月13号。

解决方法

既然普通的复制会自动把文件的修改时间设置为复制的时间，那如果能在复制文件的时候保持文件的修改时间不变就可以解决这些问题了。当然也可以尝试其他的解决方法，比如修改hexo一些配置文件的设置，让其不以文件的修改时间作为默认的博客更新时间，但是我尝试了一下没有成功，所以干脆选择了这种简单粗暴的方法——Robocopy

robocopy是windows自带的一个复制工具，它可以在复制文件的同时保留文件的时间戳，在windows的terminal中就可以使用。

robocopy D:\Document H:\Project /e

上面这行命令的意思就是将第一个文件夹中的所有文件全部复制到第二个文件夹中。
因此通过robocopy就可以实现hexo文件在不同电脑中的迁移，同时不改变文件的修改时间，从而保持博客的更新时间不变。

WSL中Rstudio-server调用mamba虚拟环境中的R

2023-11-17

前言

最近换了新的台式机，之前都是直接在服务器上linux系统中跑data。远程调用服务器上的R的话主流一般就是两种方式，一种是用vscode装插件，一种就是用Rstudio server。一般我是用mamba管理各种R和python的包，用vscode远程去跑data的，因为个人不太喜欢Rstudio。但是现在有了新的台式机，也想试试windows台式机跑data的速度，因此我本来是在windows台式机上装了mamba，并且用mamba创建了虚拟环境安装了R（之所以这么做是因为mamba或者conda管理R包或者python包真的很方便，不想在R中用install_packages/biocmanager等等去安装R包，会遇到无数的依赖问题），然后去安装了Rstudio。本来按照我的设想，因为Rstudio是可以改变R的版本的，也就是它可以更改R内核的路径，因此我就把它的路径改为了mamba虚拟环境中安装的R的路径，但是却发现Rstudio不能兼容mamba安装的R，这就很操蛋了。。。。但是我又不想放弃用mamba管理R包。
所以这就是为什么我要大费周章的在windows系统上装一个wsl，又在wsl中装一个Rstudio server，用Rstudio server来跑data。纯属是因为windows的Rstudio不支持mamba/conda安装的R包，但是我想测试一下这台windows电脑跑data的性能同时又不想放弃用mamba管理。（但其实日常的工作还是在服务器上进行的，搞这个纯属是一时兴起）。

1、安装wsl

首先就是要安装wsl，只需要用管理员身份运行PowerShell，然后输入：

wsl –install -d Ubuntu-22.04

不过在此之前需要进行一些设置，这里就不再详细写了，google一下wsl安装方法按流程走基本不会有什么问题。当然这里也可以选择不安装Ubuntu，安装其他linnux系统也可以。
下载好wsl之后，最好修改一下ubuntu的apt下载镜像，因为一开始用的是全球的镜像，下载速度可能比较慢。建议换成阿里云的镜像。
保险起见，可以首先把原来的apt镜像文件备份一下，也就是/etc/apt/sources,list这个文件。

sudo cp -v /etc/apt/sources.list /etc/apt/sources.list.backup

随后可以设置一下这个文件的权限，方便修改。

sudo chmod 777 /etc/apt/sources.list

然后就是micromamba的安装，关于micromamba/mamba/conda这里就不再详细说明，总之mamba是conda的一个替代产品，它的速度会比conda更快，而micromamba，顾名思义，就是micro的mamba。micromamba/mamba的所有命令和conda都是一样的，当然这里也可以选择使用conda或者mamba。

./bin/micromamba shell init -s bash -p ~/micromamba

3、安装Rstudio server

接下来是安装Rstudio server，这里安装的是ubuntu22的版本。安装其他版本详见：https://posit.co/download/rstudio-server/

wget https://download2.rstudio.org/server/jammy/amd64/rstudio-server-2023.09.1-494-amd64.deb
sudo gdebi rstudio-server-2023.09.1-494-amd64.deb

4、修改Rstudio server配置，使用micromamba安装的R

接下来首先用micromamba创建一个包含4.2.3版本R的虚拟环境。

micromamba create -n R4.2.3 r-base=4.2.3

随后修改Rstudio server配置文件的内容，将其调用的R的路径更换为用micromamba安装的R的路径。Rstudio server配置文件的路径应该是：/etc/rstudio/rserver.conf
用vim打开该文件，并且将文件内容修改如下：

vim /etc/rstudio/rserver.conf


上面一行是R的路径，下面一行是R的library的路径。
关于vim如何使用可以自行google。
如果不知道自己用micromamba安装的R路径在哪，可以先激活该环境，然后用which R查看路径。

micromamba activate R4.2.3
which R

5、打开Rstudio server查看效果

现在一切都配置好了，就可以打开Rstudio server看看是否成功。

sudo rstudio-server start

打开之后，在浏览器中输入：localhost:8787。输入之后按回车就可以进入Rstudio server了，8787是Rstudio server默认的端口。
进入之后，首先要输入ubuntu的账户和密码进行登录。

登录之后，就可以进入Rstudio server啦！可以看到版本号和micromamba安装的R是一致的，流程结束！现在就可以实现用micromamba随心所欲安装各种R包，管理整个R的环境了！

使用wget同时下载多个连续文件

2023-11-09

前言

今天一个朋友让我帮忙下载200多个编号连续的文件，但是一个一个手动点击下载实在是麻烦。因此就写了一个小脚本做这件事情，特此记录一下以后遇到类似的问题也可以用类似的方法解决。

适用情况

朋友让我下载的文件的编码都是连续的，如下图所示：

R脚本——生成所有文件的URL链接文件

下载文件的编号是从2023001到2023237。

a = ‘’
for(i in seq(2023001,2023237)){
　　b <- paste0(‘http://www.ppp-wizard.net/products/POST_PROCESSED/post_',i,'.tgz','\n‘)
　　a = paste0(a,b)
}
write.table(a,’a.txt’,row.names = F,col.names = F,quote = F)

这样就生成了一个每一行对应一个文件的URL下载地址的文本文件

Linux中用wget -i下载文本文件中URL对应的全部文件

wget -i a.txt

随笔杂谈

2023-11-02

2023年10月10日随笔

近日在学习深度学习的过程中脑子里突然迸发出了一些关于生命科学和深度学习模型的几点灵光，虽然这些想法可能在日后会觉得可笑，但是此时此刻，我还是想把它记录下来。

一、如何证明基因序列或者蛋白质结构的正确性？

现如今高通量测序已经是业界耳熟能详的技术了。虽然天天经受这个名词的熏陶，但是突然有一瞬间，我却对这个技术产生了陌生感。就如同有时候看一个字，这个字分明是自己再熟悉不过的一个字，但是乍一看却感觉像刚认识这个字一样陌生，不知道是否只有我才有过这种经历。而我对测序技术的陌生感来自于一个问题，既然基因序列是我们看不见摸不着的，那么我们测序得到的序列如何去证明这个序列的每一个碱基都的的确确和真实的序列相同呢？虽然测序技术的原理设计的十分巧妙，但是谁也不能完全保证这个过程完全是不出任何差错的，因为这些东西我们肉眼看不见也摸不着，在这个过程中，究竟发生了什么，谁也不知道，因此总感觉最后得到的结果有一种不真切感。就算有人说，一条序列，测许多次，就可以大大降低错误率，从而得到正确的序列，如果某个位置测几次都是相同的碱基，那么这个碱基就是一个正确的序列。但是我感觉还是无法自圆其说，基因组如此之长，每一个位置的测序错误率难道都是相同的吗，倘若有一段序列，因为序列的特殊性导致这段序列的测序错误率很高，那么就算测多次来矫正错误率，出现次数最多的还是错误的序列，反而正确的序列被掩盖了。
对蛋白质结构的也是同样的疑问。现在我们所常用的测定蛋白质结构的方法中，最先进之一的技术应该就是冷冻电镜了。但是如果我们说，用冷冻电镜测出来的结构是相当准确的，那么这个说法就需要有一个基准，那就是蛋白质的真实结构到底是什么呢？或者说，怎么能够证明，用X-ray或者冷冻电镜等方法看到的蛋白质结构就是近似接近蛋白质结构的结构呢？如果我们不知道真实的蛋白质结构是什么样的，那这些方法得到的蛋白质结构我们又是根据什么标准来说它们准确的呢？

二、训练深度学习模型也是一个永远不可能实现的永动机

按照目前我对深度学习的一些粗浅的理解，我认为一个训练好的深度学习模型本质上就是一个信息抽提机。假如说我们现在有一堆数据，如果人类直接去观察这些数据，当然也会从中获取到一些信息，但其实相对于这些数据所蕴含的全部信息而言，人类直接观察这些数据从中获取的信息是相当有限的。而一个训练好的深度学习模型，就是一个从数据中抽取信息的机器，相对于人类直接观察而言，它的效率就要高很多，甚至可能会学习到一些人类直接观察数据所得不到的信息。但是无论这个模型训练的再好，相对于数据原本蕴含的总信息量而言，它所获取的信息还是有损失的，也就是说必然有一部分信息被丢失了，谁也无法看到数据中所蕴含的全部信息的全貌。倘若深度学习模型可以学习到数据中蕴含的所有信息，实现无损压缩，那么永动机应该也可以实现了。但是这是不可能的一件事，因为谁都不知道数据中的全部信息是什么，也就是说，我们都不知道这个上限在哪里，又怎么能说我们达到了这个上限呢？

之前一直认为单细胞转录组测序中在Seurat里面经过了NormalizeData处理过的数据仅仅矫正了测序深度，而没有矫正基因长度的影响。因此NormalizeData之后的数据和TPM值是不等同的，因为TPM值是既矫正了基因长度，又矫正了测序深度。但是今天突然意识到一件事情，10X技术在建库测序的时候，对基因表达量的测定并不是依靠mapping到的片段的数量，也就是并不是靠UMI的数量来定量的，而是考UMI的种类来定量的，因此10X建库测序本来就消除了基因长度的影响。因此这里的NormalizeData之后的data数据和TPM值其实是近似等同的。

突然想到可以用深度学习做一件很有意思的事情，特别是对于生命科学而言。在生物学领域中，很多情况下是做了一些实验，发现了一些现象比如某个基因表达量上升了或者下降了还是怎样，然后猜想这个基因可能和疾病的发病机制之类的有关系。但是如果真的要找到这个基因的作用机制，就需要去查阅大量的文献去看这个基因的功能以及上下游，并且还有可能因为文献查的不够多，漏掉了这个基因曾经被报道过的一些知识，以至于错过了它导致疾病的机制的发现机会。
嗯。。。说的似乎有些混乱。也就是说，无论知识再丰富的生物学家吗，也不可能了解生物学所有的知识，比如某个蛋白某个基因是干什么的，通常很多情况下都需要搜寻文献来了解。并且，看待同样一个现象，知识渊博的人和知识匮乏的人看的东西是不一样的，也就是说，比如我看到一个基因表达量升高，并且我正好知道另外一个和这个基因相关的基因或者知识点，我就有可能把这两件事情联想在一起从而发现一个新的机制，但是如果我不知道那个知识点，我就肯定无法找到这个新的机制。因此是否我们可以通过深度学习，去训练一个模型，让它能够充分了解当今生物学所有前沿的知识点，也就是文献中报告的，包括某某某蛋白的功能，某某某基因怎么怎么样了等等等。那么它现在就是一个世界上生物学知识最丰富的“人”了，那么如果有一个新现象被观察到，那么是否它就可以根据自己已有的知识进行联想，推测出这个新现象发生的可能，当然也仅仅是可能，因为也不是所有文献报道的东西就是完全正确的。

记于2023年10月26日深圳至青岛的航班

10月26日，在前往青岛参加第十二届生物信息学大会的飞机上，读完了最近一直在看但是尚未看完的史铁生先生的散文集——《我与地坛》。读散文，不能像读一些历史性的故事书一般走马观花地读，它需要一个安静的环境，用心沉浸其中。而飞机上的时光，是一段读散文再合适不过的时间。其实想想是一件有趣的事情，或许是这万米高空让我的思绪也变得飘忽不定了吧，就好像史铁生先生坐在轮椅上在地坛中沉思的时候，一个年轻人走进了史铁生的精神世界，一个在地面上，一个在天空中，一个在若干年前，一个在若干年后，不知道史铁生先生是否在当时也想象过这样的场景，这是一种很奇幻的感觉。其实在我看来，与其说《我与地坛》是一部充满哲思的散文集，不如说是史铁生先生双腿残疾之后心灵历程的刻画，亦是史铁生先生自我拯救或者说自我对话的过程。其中，“好运设计”一段令我感触最深，下面截取这一部分的部分原文。

“阴影最初是这样露头的：你能在一场如此称心、如此顺利、如此圆满的爱情和婚姻中饱尝幸福吗？也就是说，没有挫折，没有坎坷，没有望眼欲穿的企盼，没有撕心裂肺的煎熬，没有痛不欲生的痴癫与疯狂，没有万死不悔的追求与等待，当成功到来之时你会有感慨万端的喜悦吗？在成功到来之后还会不会有刻骨铭心的幸福？或者，这喜悦能到什么程度？这幸福能被珍惜多久？会不会因为顺利而冲淡其魅力？会不会因为圆满而阻塞了渴望，而限制了想象，而丧失了激情，从而在以后漫长的岁月中是遵从了一套经济规律、一种生理程序、一个物理时间，心路却已荒芜，然后是腻烦，然后靠流言蜚语排遣这腻烦，继而是麻木，继而用插科打诨加剧这麻木——会不会？会不会是这样？地球如此方便如此称心地把月亮搂进了自己的怀中，没有了阴晴圆缺，没有了潮汐涨落，没有了距离便没有了路程，没有了斥力也就没有了引力，那是什么呢？很明白，那是死亡。当然一切都在走向那里，当然那是一切的归宿，宇宙在走向热寂。但此刻宇宙正在旋转，正在飞驰，正在高歌狂舞，正借助了星汉迢迢，借助了光阴漫漫，享受着它的路途，享受着坍塌后不死的沉吟，享受着爆炸后辉煌的咏叹，享受着追寻与等待，这才是幸运，这才是真正的幸运，恰恰死亡之前这波澜壮阔的挥洒，这精彩纷呈的燃烧才是幸运者得天独厚的机会。你是一个幸运者，这一点你要牢记。所以你不能学凡夫俗子的梦想，我们也不能满意这晴空朗日水静风平的设计。所谓好运，所谓幸福，显然不是一种客观的程序，而完全是心灵的感受，是强烈的幸福感罢了。幸福感，对了。没有痛苦和磨难你就不能强烈地感受到幸福，对了。那只是舒适只是平庸，不是好运不是幸福，这下对了。”

“幸福感，对了。没有痛苦和磨难你就不能强烈地感受到幸福，对了。那只是舒适只是平庸，不是好运不是幸福，这下对了。”，读完这段话之后，仿佛被锤子击打了心弦一般。人人生而不同，这种不同体现在方方面面，不仅仅是自身的不同，还包括家境等外在条件的不同。但是正如我们没道理抱怨别人生下来就比自己智商高一样，我们也不应该去抱怨别人的家境比自己好，这些外界条件和智商等自身条件一样，都是与生俱来的，所以有很多人仇富，或者说有一种不平衡的心理，我是不能苟同的。况且，我也并不认为家境优越的人就是幸福的，大家不过是各有各的烦恼罢了，我也并不羡慕他们，正如史铁生先生所说的，“那只是舒适只是平庸，不是好运不是幸福”。这并非是我吃不到葡萄说葡萄酸。举一个简单的例子吧，一个很好的朋友小A，家庭资产，姑且估摸着有几个亿吧，但是他是一个欲望很低的人，总给人一种什么都无所谓的样子。这一辈子就算再怎么样，也很难超越父辈的成就了。这也就是史铁生先生所描绘的，当一切都顺利的时候，是很难感受到幸福的，因为幸福的阈值会升高，感受到的不是幸福，而只能称之为舒适。
因此，看吧，幸福对所有人都是公平的，不会因为有钱就会感到幸福，也不会因为穷苦感受不到幸福，大家各自有各自的烦恼。幸福，其实就像是在玩捉迷藏的一个调皮的孩子，当你找到它的时候，它就会马上藏起来。那么，如果一找到幸福它就马上藏起来，幸福真的就是无法获得无法捕捉的吗？其实仔细想想，我们好像从来没有真正的得到幸福，但是真正的原因是，幸福已经无处不在了，它就蕴含在我们追求幸福的过程中。正如张爱玲所说，“人们总是在接近幸福时倍感幸福，在幸福进行时患得患失”，我想，大抵也是在说同一件事情。
所以，幸福就存在于我们追求幸福的过程中，只不过我们身在局中而不自知，左手拿着幸福，右手寻找幸福。

UKBiobank数据提取

2023-10-15

表型数据提取教程

注：本文所使用的一些原始文静是通过R包ubktools生成的，生成这些文件的详细方法见：
https://kenhanscombe.github.io/ukbtools/articles/explore-ukb-data.html
这里不再赘述
1、UKBiobank表型数据目录文件简述

“ukb50136.html”这个文件是UKBiobank表型数据的目录文件，它是通过ukbtools这个R包生成的一个网页文件。文件中存储了UKB表型数据的信息，包括这些表型数据的UDI号、所包含的人数、数据类型以及表型数据的描述等信息。

可以看到这个html文件中是一个表格，包含五列。分别是Column、UDI、Count、Type、Description。Column这一列代表了数据的行数。UDI是每一行数据的一个特有的标识号，其中同一表型数据的UDI号“-”之前的部分是相同的，在提取表型数据的时候也是根据这个UDI号来提取的。Type表示这个表型数据的数据类型，有整型、文本型等。Description就是对这个表型数据信息的描述。
并且从图片左上角可以看到，这个表格的总行数是20541，因此所拥有的表型信息是非常丰富的，可以自行浏览这个表型数据目录文件，决定自己需要什么样的表型数据。

2、提取数据所需要的文件

①.enc_ukb文件（我们这里应该是ukb50136.enc_ukb）：这个文件中存储了所有的表型数据，我们也就是要从这个文件中提取我们所需要的表型数据。这个文件是通过ukb_unpack解密ukbxxxx.enc文件生成的，因为已经生成了.enc_ukb文件，直接使用就可以了，这里不再赘述生成这个文件的方法。

②ukbconv文件：这个是UKB官方所提供的一个工具文件，可以在UKB官网上下载。而在这里我们也已经准备好了这个文件，具体下载方法可以自行浏览UKB官网。

③MY.fields.id文件：也就是一个名为MY.fields.id的文本文件（当然这个文件名可以自己定义，自己进行修改）。这个文本文件中只含有一列数据，也就是想要提取的表型数据的UDI号（可以通过查看ukb50136.html这个表型数据目录文件获得），注意：这里的UDI号是UDI“-”之前的整数部分。也就是说，如下图所示，如果想要提取“Verbal interview duration”这个表型数据，只需要在这个文本文件中输入3就可以了。如果想要同时提取多个表型数据，就将想要提取的表型数据的UDI号写成一列存放在MY.fields.id文本文件中。

比如在这里，提取的就是Verbal interview duration、Biometrics duration、Sample collection duration、Conclusion duration、Heel ultrasound method等五个表型信息。

3、提取数据

准备好了以上三个文件之后，就可以进行数据的提取了（注意这三个文件要在同一个目录下）。但是命令的执行需要在linux系统下，如果没有linux系统则可以下载wsl2作为替代。如果是在wsl2也就是windows subsystem for linux下进行操作的话，磁盘的挂载点应该在/mnt目录下，也就是原来的windows电脑中的各个磁盘都在/mnt目录下。比如D盘的路径就是/mnt/d，如果是外接硬盘，也应该是在/mnt路径下。因此如果是直接在外接硬盘中对数据进行提取，那么进入wsl之后输入命令“cd /mnt/xxx”进入到外接硬盘的目录下，然后再输入”cd xxx”将工作目录切换到存储了三个准备文件的目录中。
切换好工作目录之后，输入命令：
ukbconv ukb50136.enc_ukb r -iMY.fields.id -oMY
执行结束后，会生成一个“MY.tab”的文件，提取的表型数据就在这个文件中了。当然这个输出文件的名字可以自己修改，也就是将原命令最后的MY修改成所想要的文件名就可以了。MY.tab文件可以通过excel等打开。

Autodock对接结果分析

2023-02-28

Autodock对接结果分析

1、用AutodockTools分析结果

点击Analyze——Docking——Open Autodock vina result，打开生成的pdbqt类型的对接结果文件

打开之后可以用键盘的左右方向键切换结果包含的几个pose，同时还会显示每个pose的对接分数
如果想展示受体蛋白和小分子对接之后的相互作用，可以把受体蛋白也加载进来，点击Analyze——Macromolecule——Open，打开受体蛋白的pdbqt文件

然后点击Analyze——Docking——Show interactions

2、生成某个pose的小分子与蛋白质的pdb复合物文件

在ADT中是无法将蛋白质和小分子保存到同一个文件里面的，但是可以保存对接后小分子的某一个pose到PDB文件里
首先打开对接的结果文件，用键盘的左右方向键切换pose，切换到想要保存的pose，然后点击File——Save——write PDB

这样就把pdbqt格式的结果文件中的某一个pose的小分子保存为了pdb文件

然后就可以利用Pymol把小分子的pdb文件和蛋白质的pdb文件合并为一个pdb文件
首先在Pymol中打开之前保存的某一个pose的小分子pdb文件以及蛋白质pdb文件（一起打开），一起打开后小分子就会结合到蛋白质上。
然后点击File——Export Molecule，就可以进行保存了，在保存的时候要把小分子pdb文件和蛋白质pdb文件同时勾选上，才能将它们保存到同一个pdb文件中。

这样就保存好了某一个pose的小分子和蛋白质的复合物pdb文件

此外，也可以直接使用Pymol生成对接之后某个pose的小分子和蛋白质的复合物pdb文件，不需要先使用ADT生成对接后某个pose的小分子pdb文件
Pymol是可以直接识别pdbqt这个类型的文件的，因此可以直接把pdbqt格式的结果文件打开。打开之后，屏幕右下角state会显示小分子pose的数量以及当前显示的是第几个pose

可以按播放按钮来查看其他的pose
如果播放速度太快的话，可以点击Movie——Frame Rate来设置播放速度，默认是30FPS

把蛋白质pdb文件加载进来，就可以保存蛋白质和某一个pose的小分子的复合物pdb文件，同样，切换成其他pose就可以保存其他pose的小分子和蛋白质的复合物pdb文件

除了使用Pymol来生成小分子和蛋白质的复合物pdb文件之外，也可以使用UCSF chimera来处理

点击Tools——Surface/Binding Analysis——ViewDock，打开vina对接的pdbqt的结果文件
打开之后会显示如下界面

此时，将蛋白质的pdb文件也加载进来，打开之后，蛋白质和小分子其实是在不同的model里面，因此需要把这两个model合并一下，然后再保存到一个pdb文件里面
点击Favorites——Model panel

点击之后会出现一个新的面板

可以看到现在有两个model，现在需要把两个model放到同一个model里面，点击右侧的copy/combined

点击之后会出现一个新面板

前九个为不同pose的小分子，最后一个为蛋白质，按住ctrl键然后用鼠标选择蛋白质和任意一个pose的小分子，选好之后点击apply，模型面板中就会变成三个model

现在可以选中蛋白质和小分子的model，点击close将它们关闭，然后就只剩下了小分子和蛋白质的复合物model，然后就可以保存为pdb文件了，File——Save PDB
